AssessmentEvent Class

This class describes the performance of an assessment of a particular aspect of a patient’s condition. Such assessments employ a test or instrument, often a questionnaire, that quantify otherwise subjective observations. These instruments employ scoring sysystems which have been systemetized and validated. Standardized assessment tools create scores and (sometimes) sub-scores which are then used to determine the optimal placement and treatment plans for the patient and to track patient progress over time. Standardized assessment tools facilitate the collection of accurate information in a reliable and consistent manner, provide clinicians and treatment agencies with increased accountability, identify patient progress over time, and provide a basis to adjust treatment plans.

AssessmentInstrumentQuestion(Abstract) Class

This class represents a question on an Assessment Instrument as well as informatin regarding the possible response to that question. Note that the response might be entered by an interviewer, entered by the interviewee, or populated by a software system (e.g., a system could automatically populate a question such as Date Created). Note also that this class defines both the question, which is the same for every instance of an edition of a questionaire, and the response which is populated for each administration of the instrument. For a given Assessment Instrument, the question part would be hard-coded while the response would be populated at run-time.

AssessmentInstrumentQuestion(Abstract).assessmentQuestion

This property uniquely identifies the question being asked within the Assessment Instrument. This is treated as a code because the Name and Version of the Assessment Instrument plus the Question Number serves as an identifier, while the text of the question can be represented as the associated text. For example, the Addiction Severity Index, 5th Edition question G.4 is the "Date of Admission".

AssessmentInstrumentQuestion(Abstract).assessmentQuestionId

This property is used to convey the question "number" within the Assessment Instrument. While the assessmentQuestion uniquely identifies the question across all possible Assessment Instruments, it does not necessarily contain the question number as displayed on the Assessment Instrument. This property therefore is needed to hold that data.

AssessmentInstrumentQuestion(Abstract).possibleResponse

This property represents the possible responses to a question on an Assessment Instrument. It defines the permissible data type, and if the question is a multiple choice, will define the list of valid responses by using an enumeration. Note that this property is optional (i.e., cardinality is 0..1), because clinical records will point to the response, which in turn points to the question. In this case, the list of possible responses is not needed; the actual response is already known and is all that is needed.

AssessmentResponse Class

This class represents a response to an Assessment Instrument question upon the administration of the Assessment Instrument. It contains a pointer to the question as well as the response itself. Note that the response might be entered by an interviewer, entered by the interviewee, or populated by a software system (e.g., a system could automatically populate a question such as Date Administered).

SubScore Class

This class represents the total of the values associated with a defined subset of the Assessment. Some Assessment Instruments define a intermediate score(s) for related characteristics along with a total score for the overall instrument. For example, an overall mental health Assessment instrument may define a sub-score for depression. Note that many Assessment Instruments contain groupings or sections, which are not necessarily related to sub-scores.
