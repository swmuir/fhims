Entity(Abstract).id

An identifier that uniquely identifies the individual. Note that this property can hold more than one value, and that the datatype contains an indicator of the kind of identifier that is assigned. For example, a Person record might contain an SSN, a driver's license number, and a locally-assigned id. These identifiers, while different, all identify the same individual. However, the FHIM calls out certain Ids that are specifically required. For example, the Person class explictly contains an SSN property. In the case when an Id is explicitly modeled, that Id type should not be included in this property. Therefore, an SSN should not be instantiated in this property, because a separate SSN field exists for that purpose.

"This field contains the list of identifiers (one or more) used by the healthcare facility to uniquely identify a patient (e.g., medical record number, billing number, birth registry, national unique individual identifier, etc.)." - HL7 Version 2.8, PID-3

"Identifier for a person within a particular scope." - HL7 FHIR, Person.identifier.

A group of people whose members identify with each other, through a common heritage, often consisting of a common language, a common culture (often including a shared religion) and an ideology that stresses common ancestry or endogamy .
